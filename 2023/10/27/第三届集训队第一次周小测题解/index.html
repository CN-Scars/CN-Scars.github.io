题目1: 哪里贵了

题目描述

在一次直播销售活动中，消费者在弹幕中提出“这个品牌越来越贵了”时，销售员里加其（Rigachi）回应说：“哪里贵了？”。为了解决这个问题，他决定为每一种商品打一个折扣。然而，他只想为那些定价过高的商品打折。里加其（Rigachi）认为，如果一个商品的价格超过了所有商品的平均价格，那么这个商品就定价过高。定价过高的商品应该打9折，其他商品的价格保持不变。现在，请你来帮他计算出每种商品的折扣后的价格。

输入格式:

输入共有2行
第一行，输入商品的数量$n$ （$1 \leq n \leq 10$）
第二行，$n$个浮点数$a_i$，$a_i$表示每种商品的价格（$1 \leq a_i \leq 2^{31} - 1$）

输出格式:

$n$个浮点数，代表每种商品的折扣后的价格（保留至小数点后1位），中间以一个空格分割，末尾不得有多余的空格

输出样例:

解题思路

  1. 计算平均价格:

    • 首先需要确定所有商品的总价。通过遍历商品价格并累加它们来完成。
    • 然后，将总价除以商品数量，以得到商品的平均价格。

  2. 确定哪些商品需要打折:

    • 遍历商品价格列表。对于列表中的每个价格，判断它是否超过平均价格。
      • 如果是，将此商品价格乘以 0.9（即打9折）。
      • 否则，保持价格不变。

  3. 输出处理:

    • 按照题目的要求，输出每个商品的折扣后价格，并确保价格保留到小数点后一位。在输出时，确保价格之间有一个空格分隔，并且最后一个价格后没有多余的空格。

题目2: 蜂蜜

题目描述

虫大和金发妹在森林中散步时发现了一罐蜂蜜，他们都想把蜂蜜送给翠花，所以他们决定以石头剪刀布决定谁能将蜂蜜送给翠花。已知他们的出拳都是有周期性规律的，比如：“石头-布-石头-剪刀-石头-布-石头-剪刀……”，就是以“石头-布-石头-剪刀”为周期不断循环的。请问，虫大和金发妹比了N轮之后，谁能获得这罐蜂蜜？

输入格式:

输入包含三行。
第一行包含三个整数：N，NA，NB，分别表示比了N轮，虫大出拳的周期长度，金发妹出拳的周期长度。$ 0 < N, N_A, N_B < 100 $。
第二行包含NA个整数，表示虫大出拳的规律。
第三行包含NB个整数，表示金发妹出拳的规律。
其中，0表示“石头”，2表示“剪刀”，5表示“布”。相邻两个整数之间用单个空格隔开。

输出格式:

输出一行，如果虫大赢的轮数多，输出虫大；如果金发妹赢的轮数多，输出金发妹；如果两熊打平，输出===。

输出样例:

样例解释

轮数 虫大出拳 金发妹出拳 本轮结果
0 0 0 平局
1 2 5 虫大胜

4 2 0 金发妹胜
5 5 5 平局

最终结果：虫大4分，金发妹2分，虫大胜

解题思路

  1. 读取输入数据:

    • 读取N轮比赛，以及虫大和金发妹的出拳规律的周期长度NA和NB。
    • 读取虫大的NA次出拳规律。
    • 读取金发妹的NB次出拳规律。

  2. 初始化得分:

    • 创建两个变量，一个用于跟踪虫大的得分，另一个用于跟踪金发妹的得分。

  3. 比较出拳:

    • 对于从0到N-1的每一轮，按照以下步骤进行：
      • 使用取模运算符%来确定虫大和金发妹在当前轮次应该使用哪个出拳。这样可以确保他们的出拳是周期性的。
      • 根据两人的出拳决定这一轮的胜者。
        • 如果他们的出拳相同，那么这一轮是平局。
        • 根据“石头-剪刀-布”的规则，确定赢家，并给赢家加1分。

  4. 输出结果:

    • 在所有轮次之后，根据两人的得分输出胜者。
      • 如果虫大的得分高，输出”虫大”。
      • 如果金发妹的得分高，输出”金发妹”。
      • 如果两人得分相同，输出”===”。

题目3: PIN码输入

题目描述

输入一个四位数的PIN码，由1, 2, 3, 4, 5, 6, 7, 8, 9和0构成。

上图显示了用于输入 PIN 码的设备的屏幕。最初，光标指向数字1，在一秒钟内，你可以执行以下两个操作之一：

  • 选择当前光标所指的数字作为一位PIN码
  • 将光标移动到与当前位置相邻位置的数字

例如，对于数字5，在一秒钟内，就可以选择5作为PIN码的下一位，也可以将光标移动至相邻的数字，即为4和6，对于数字0，只有一个相邻的数字，9。

请你编写代码，确定输入给定的四位数 PIN 码所需的最小秒数。

输入格式:

一行长度为4的只由1, 2, 3, 4, 5, 6, 7, 8, 9和0构成的PIN码

输出格式:

一行，输入给定的四位数 PIN 码所需的最小秒数。

样例解释:

  • 在第一个测试用例中，需要按下光标所指的数字（1）4次。
  • 在第二个测试用例中，完成输入所需的9秒的操作步骤如下：
    按下光标所指的数字（1）。
    将光标移动到数字2
    按下光标所指的数字（2）。
    将光标移动到数字3

    将光标移动到数字4
    将光标移动到数字5
    将光标移动到数字6
    按下光标所指的数字（6)。

解题思路

  1. 初始化：首先，我们从数字1开始，因为光标开始时是指向1的。所以设置初始位置为1。

  2. 转换：对于数字0，我们将其视为10来进行处理，因为在我们的键盘布局中，数字0实际上是处于数字9之后的。

  3. 遍历PIN码：对于PIN码中的每一位数字，做如下处理：

    a. 计算距离：确定当前数字与前一数字的距离。这里的”距离”是指的从前一个数字移动到当前数字所需要的最少秒数（即最少步数）。因为键盘是线性的，所以这个距离就是两个数字的绝对值差。

    b. 更新总秒数：到目前为止，我们需要的秒数是距离+1。这里加1是因为我们还需要额外的1秒来按下光标所指的数字。因此，更新总秒数为ans += 距离 + 1。

    c. 更新当前位置：将当前数字设置为新的位置，为下一次迭代做准备。

  4. 输出：当我们处理完PIN码中的所有数字后，输出总的所需秒数。

这种方法的优点是它简洁、直观，并且有很高的效率。它仅需线性时间来遍历PIN码，并且不需要额外的存储空间。

题目4: 只因你太美

题目描述

设有一个正整数 $a$。我们将 $a$ 分解为$n$个质数的乘积。

令 $a = p_1 \times p_2 \times p_3 \times \dots \times p_n$，其中每个 $p_i$ 均为质数。

当且仅当 $p_i$ 都是奇数时，我们称 $a$ 为只因数。对于给定的一个正整数m，如果m为只因数，输出“只因你太美”的英文Just because you are so beautiful，接着进行判断，若该只因数最大的分解结果就是该只因数本身，输出!!!，否则输出!。若m不是只因数，则只输出一行Just because you are not beautiful

输入格式:

第一行是一个整数，表示数据组数 $T$。

接下来 $T$ 行，每行一个整数，表示一组数据的 $n$。

输出格式:

对每组数据，如果 $n$ 是『只因数』，请输出Just because you are so beautiful，若该只因数最大的分解结果就是该只因数本身，输出!!!，否则输出!。若m不是只因数，则只输出一行Just because you are not beautiful

数据规模与约定

本题共有 $5$ 个测试点。

对 $100%$ 的数据，保证$1 \leq T \leq 10$，$2 \leq n \leq 10^{18}$。

解题思路

  1. 判断“只因数”条件：为了确定一个数是否是“只因数”，我们需要确保该数的所有质数因子都是奇数。事实上，除了数字2之外的所有质数都是奇数。因此，只要一个数是奇数，它就可能是“只因数”。这是因为任何偶数都会被2整除，这意味着它至少有一个偶数质数因子（即2）。所以我们可以简单地通过检查数是否是奇数来判断它是否是“只因数”。

  2. 检查是否为质数：为了确定一个数字是否是它自己的最大“只因数”，我们需要检查它是否是质数。这是因为质数只能被1和它自己整除，所以它不可能有其他的质数因子。

  3. 算法步骤：

    • 首先，读取测试案例的数量。
    • 对于每一个测试案例：
      • 检查是否是“只因数”（也就是检查该数是否是奇数）。
      • 如果是，“只因数”，则进一步检查是否为质数。
      • 根据上述检查的结果，输出相应的消息。

可能出现全为空格的字符串和字符。

输入格式:

测试数据有多组，每组输入字符串$s$和字符$c$。

输出格式:

解题思路

  1. 读取输入：读取要处理的字符串以及需要从中删除的字符。
  2. 字符处理：遍历整个字符串，检查每个字符。如果该字符不是要删除的字符，则保留该字符。
  3. 输出结果：输出处理后的字符串。如果处理后的字符串为空，输出NULL。

对于C：

  1. 使用gets函数获取字符串，然后用getchar来读取要移除的字符。为了确保下次循环可以正确读取s，读取一个字符（使用getchar()）来读取换行符后面的字符。
  2. 设置一个flag变量来检查是否输出了除c以外的任何字符。遍历s中的每个字符，如果当前字符不是c，则输出该字符并将flag设置为1。
  3. 根据flag的值来决定是否输出NULL。每次处理完毕后，输出一个换行符，以隔开每次的输出。

对于C++：

  1. 使用getline函数从标准输入读取字符串。然后再次使用getline函数从标准输入读取一个字符串，并从该字符串获取要移除的字符。

  2. 使用vector<char>数据结构来存储除c之外的所有字符。遍历s的每个字符，如果当前字符不是c，则将其添加到ans向量中。

  3. 如果ans向量不为空，则输出ans中的所有字符。如果ans是空的，输出NULL。每次处理完毕后，输出一个换行符，以隔开每次的输出。

    注：本题C++语言解法和C语言解法的差距仅限于数据的读取，C++语言在处理数据时也可采用一个flag标记变量的方法来实现需求。
